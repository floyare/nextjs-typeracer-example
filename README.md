# Next.js Typeracer Example

A real-time writing competition platform built for the Next.js Coding Challenge. Players can join a lobby, race against each other in real-time, and see live progress, words per minute (WPM), and accuracy stats.

## Tech Stack & Architecture Choices

- **Frontend**: Next.js 15 (App Router), React 19, TypeScript, Tailwind CSS.
- **Realtime API & Database**: [Convex](https://www.convex.dev/). 
  - *Why Convex?* The challenge required a real-time API. Convex provides a fully-managed, end-to-end type-safe real-time database with built-in WebSocket syncing. It removes the massive overhead of manually managing WebSocket connections (e.g., Socket.io), handling reconnections, or setting up Redis for pub/sub. It integrates flawlessly with Next.js and React via `useQuery`/`useMutation` hooks.
- **Styling & UI**: Tailwind CSS for rapid styling, `lucide-react` for icons, and some `radix-ui` primitives for accessible UI components.
- **Testing**: Vitest (Unit/Integration), Playwright (E2E).

## Features Implemented

1. **Real-time Multiplayer Integration**: Live updates of player progress, WPM, and accuracy directly in the UI.
2. **Fixed-time Rounds**: Races automatically start and end based on server-side timestamps.
3. **Live Metrics**: Real-time calculation of WPM (only counting correctly typed words) and Accuracy percentage.
4. **Session Persistence & Anti-spam**: Uses `localStorage` combined with backend validation to prevent a single user from opening multiple tabs and creating "ghost players" in the same room.
5. **Robust Room Management**: Automatic scheduled cleanup of empty game rooms and inactive players (heartbeat mechanism).

## How to Run

### 1. Install Dependencies
```bash
pnpm install
```

### 2. Set up Convex
Since this uses Convex as the backend, you need to initialize the dev environment. This will prompt you to log in and create a project, automatically setting up your `.env.local`.
```bash
pnpx convex dev
```

### 3. Start the Next.js Development Server
In a separate terminal window:
```bash
pnpm run dev
```
Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Testing

This project includes a comprehensive testing suite simulating a production-ready environment.

```bash
# Run Unit, Component, and Backend (Convex) tests
pnpm run test

# Run End-to-End (E2E) tests via Playwright
pnpm run test:e2e
```

## Future Improvements (Production Next Steps)

If this were a real production application, I would implement the following next steps:

1. **Authentication**: Replace the simple `localStorage` session ID with a robust authentication provider like Clerk, NextAuth, or Convex Auth to track user stats permanently across devices.
2. **Global Leaderboard & Database Indexing**: Add pagination and database indexes (`by_wpm`, `by_wins`) to support a global historical leaderboard.
3. **Advanced Anti-Cheat**: Implement server-side validation of keystroke timings and intervals to prevent users from simply copy-pasting the text or using bots.
4. **External API Integration**: Fetch sentences or quotes from a reliable external API (like a quote generator) instead of relying on a hardcoded list, adding variety to the races.
5. **Enhanced UI/UX**: Improve the overall user interface and experience, including adding smooth racing animations (similar to the original Typeracer).
6. **Code Optimization**: Further refactor the codebase to ensure functions remain DRY and highly maintainable.
7. **Comprehensive Error Handling**: Enhance global error handling and provide better feedback to the user for edge cases.
